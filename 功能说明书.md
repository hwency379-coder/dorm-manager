# 宿舍极简物品管理系统 - 功能说明书

## 一、系统概述

本系统是一个面向大学生宿舍场景的个人物品管理工具，采用极简设计理念，帮助用户对宿舍内所有物品进行功能分区管理、收纳位置标记、使用状态追踪和定期复盘清理。

**核心定位**：每一件东西都有明确功能，该留的留、该丢的丢。

---

## 二、三大页面总览

### 2.1 功能页（底部导航：功能）
按物品用途分为 8 大分区，点击分区查看该类别下的所有物品。

| 分区 | 说明 | 专属字段 |
|------|------|---------|
| 衣装 | 衣物鞋帽配饰 | 品牌、尺码、颜色、季节、洗护方式、材质 |
| 食饮 | 食品饮品调料 | 保质期、数量、囤货阈值、储存方式 |
| 书籍 | 教材课外书笔记本 | 作者、出版社、是否教材、相关课程 |
| 杂物 | 收纳箱文具药品 | 规格、数量 |
| 休憩 | 床品抱枕玩偶 | 尺寸、材质、可拆洗 |
| 电子 | 数码配件充电线 | 品牌、型号、保修期 |
| 日用 | 洗漱清洁纸巾 | 数量、囤货阈值、规格 |
| 待处理 | 闲置待转让丢弃 | 原价、处理方式、意向接手人 |

**操作**：点击任意分区卡片 → 查看该分区物品列表 → 点击具体物品编辑 → 右下角悬浮 + 按钮添加新物品（自动归入当前分区）。

---

### 2.2 首页（底部导航：首页）
提供两种视图模式，通过顶部分段器切换。

#### 2.2.1 地图视图
以宿舍平面图为底图，在图上标记收纳点位置。

| 功能 | 操作方式 |
|------|---------|
| 上传平面图 | 首次进入点击"选择图片"上传宿舍平面图（JPG/PNG） |
| 添加收纳点 | 点击平面图上任意空白位置 → 输入收纳点名称（如"衣柜上层"）→ 保存 |
| 查看收纳点详情 | 点击平面图上的标记图标或下方列表中的收纳点 |
| 上传实拍图 | 进入收纳点详情 → 点击"上传实拍图"可上传该空间的真实照片 |
| 添加子收纳点 | 进入收纳点详情 → 点击"添加子收纳点" |
| 进入子层 | 点击收纳点详情中的子收纳点 → 面包屑导航会显示层级路径 |
| 添加物品 | 右下角悬浮 + 按钮 → 打开物品表单 |
| 删除收纳点 | 收纳点详情底部"删除收纳点"按钮（会同时删除所有子收纳点） |
| 重新上传图片 | 底部操作栏"重新上传"按钮（会保留所有收纳点位置） |
| 面包屑导航 | 顶部显示如"宿舍 / 衣柜 / 上层"，点击任意层级可跳回 |

**嵌套结构**：宿舍平面图 → 收纳点（如衣柜）→ 子收纳点（如上层、下层），最多支持无限层级嵌套。

#### 2.2.2 表格视图
以列表形式展示所有物品，支持搜索和多条件筛选。

| 功能 | 操作方式 |
|------|---------|
| 搜索 | 点击顶部搜索图标 → 输入物品名称关键词 |
| 筛选 | 点击"筛选"按钮 → 按分区/状态/频率/更多条件组合筛选 |
| 快捷标签 | 筛选后在筛选栏显示已选条件标签，点击标签上的 X 可单独移除 |
| 筛选维度 | 分区（8类）、状态（在用/待处理/已处理）、频率（每日~极少）、二手、有照片、囤货预警 |

**物品卡片信息**：照片缩略图（或分区色块）+ 名称 + 状态标签 + 使用频率 + 数量 + 二手标记 + 囤货预警标记。

---

### 2.3 复盘页（底部导航：复盘）
提供数据概览和管理工具。

#### 2.3.1 资产卡片
- 显示所有在用/待处理物品的标价总价值
- 按分区展示彩色比例条（衣装红色、食饮黄色、书籍蓝色...）
- 点击卡片 → 查看资产明细（每件物品的名称、品牌、原价）
- 已处理物品不计入资产统计

#### 2.3.2 关注卡片
- 自动计算需要关注的物品数量
- **库存不足**：数量低于囤货阈值的日用品/食品（红色）
- **待处理**：状态为"待处理"的物品（橙色）
- 点击卡片 → 查看具体关注清单 → 点击物品可直接编辑

#### 2.3.3 温度速查
输入当前温度，查看匹配的穿搭套装。

| 功能 | 说明 |
|------|------|
| 套装模式 | 显示所有已保存的套装，输入温度后自动筛选匹配套装 |
| 单件模式 | 网格展示所有在用衣装的照片 |
| 添加套装 | 输入套装名称、温度区间（最低~最高°C）、选择组成衣装、上传套装照片 |
| 编辑套装 | 点击套装卡片上的铅笔图标 |
| 删除套装 | 点击套装卡片上的垃圾桶图标 |

#### 2.3.4 已处理记录
- 列出所有状态为"已处理"的物品
- 按处理时间倒序排列
- 点击物品可查看详情或重新编辑

#### 2.3.5 收纳点清单
- 树状展示所有平面图及其收纳点层级结构
- 点击平面图名称可跳转至首页地图视图

#### 2.3.6 物品概况
- 总物品数 / 在用数 / 已处理数 三个统计指标

## 三、物品表单（添加/编辑）

### 3.1 基础字段
| 字段 | 必填 | 说明 |
|------|------|------|
| 物品名称 | 是 | 物品的名称 |
| 功能分区 | 是 | 8大分区之一 |
| 所在位置 | 否 | 关联到具体收纳点（从地图视图中选择） |
| 使用状态 | 是 | 在用 / 待处理 / 已处理 |
| 使用频率 | 是 | 每日 / 每周 / 每月 / 换季 / 极少 |
| 品牌 | 否 | 品牌或店铺名 |
| 数量 | 否 | 当前持有数量 |
| 囤货阈值 | 否 | 低于此数量时触发预警提醒 |
| 备注 | 否 | 任意补充说明 |

### 3.2 分区专属字段
根据所选分区自动显示不同字段（如衣装显示尺码/颜色/季节，食饮显示保质期/储存方式等）。

### 3.3 照片
- 支持上传多张物品照片（调用设备相机或相册）
- 点击已上传照片上的 X 删除

### 3.4 二手/可转让标记
- 开关开启后显示额外字段：原价（元）、购买凭证照片
- 方便后续转让时提供购买证明

## 四、数据管理（设置面板）

点击首页右上角设置图标进入。

| 功能 | 说明 |
|------|------|
| 导出 JSON 备份 | 将所有数据（物品、平面图、套装）导出为 JSON 文件下载到本地 |
| 从 JSON 导入 | 选择之前备份的 JSON 文件恢复数据 |
| 微信文本导出 | 生成格式化的文字清单，自动复制到剪贴板，可直接粘贴到微信 |
| 清空所有数据 | **危险操作**，删除所有物品、平面图、套装数据（不可逆） |

## 五、数据存储说明

### 5.1 存储方式
所有数据保存在浏览器本地（localStorage），**不上传到任何服务器**。

| 存储键 | 内容 |
|--------|------|
| `dorm_v3` | 物品列表 + 平面图数据 + 最后同步时间 |
| `dorm_outfits` | 套装列表 |

### 5.2 数据持久性
- **同一台设备同一浏览器**：数据永久保存，关闭页面/关机再打开仍然存在
- **更换浏览器**：数据不会同步（Chrome 的数据 Edge 看不到）
- **清理浏览器缓存**：数据会被清除
- **存储上限**：约 5MB，大量高清照片可能触发上限

### 5.3 首次使用
系统预置了 20 条演示数据（涵盖 8 个分区），首次打开自动加载。可通过"清空所有数据"删除演示数据后重新开始。

## 七、使用建议

### 7.1 初始设置流程
1. 进入首页 → 地图视图 → 上传宿舍平面图
2. 在平面图上标记主要收纳点（衣柜、书桌、床底收纳箱等）
3. 为每个收纳点上传实拍图（方便快速辨认）
4. 通过右下角 + 按钮或功能页逐条添加物品

### 7.2 日常维护
- **用完即记**：新买东西或处理掉旧东西时立即更新状态
- **定期复盘**：每周打开复盘页查看"需要关注"卡片，补充库存或处理闲置
- **换季整理**：温度变化时使用"温度速查"功能快速找到合适穿搭

### 7.3 数据安全
- 重要变更后建议导出 JSON 备份到本地
- 换设备时通过 JSON 导出/导入迁移数据
- 微信文本导出可用于向室友/家人分享物品清单

## 八、技术架构

| 项目 | 说明 |
|------|------|
| 前端框架 | 纯原生 JavaScript（无 React/Vue 等框架依赖） |
| 样式方案 | Tailwind CSS（CDN 引入）+ 自定义 CSS |
| 文件结构 | `index.html` + `style.css` + `script.js` 三个文件 |
| 数据存储 | localStorage（浏览器本地存储） |
| 图标方案 | 内联 SVG（不依赖外部图标库） |
| 兼容性 | 现代浏览器（Chrome/Safari/Edge/Firefox） |
